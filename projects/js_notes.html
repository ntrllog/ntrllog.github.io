Let vs. Var

var a = "a";
var a = "b";

No error because the value of a will be overwritten.

let a = "a";
let a = "b";

Will cause an error because a is declared twice.

for (var i = 0; i < 3; i++) {
}
console.log(i); // will print 3

Variables declared with var are global (but local if declared inside a function).

Variables declared with let are local to its scope.

Class Syntax

class Book {
  constructor(author) {
    this._author = author;
  }

  get writer() {
    return this._author;
  }

  set writer(updatedAuthor) {
    this._author = updatedAuthor;
  }
}

const lol = new Book('anonymous');
console.log(lol.writer);  // anonymous
lol.writer = 'Big Tasty';
console.log(lol.writer);  // Big Tasty

class declares a new function. The constructor is called when new is called.

Object Mutation

Const

const a = [2, 3, 5];
a = [2, 4, 6]; // error
a[0] = 1; // no error

const prevents variables from being reassigned. But the object they're pointing to can change.

Object.freeze

const o = {
    name: 'Bob'
}

Object.freeze(o);
o.name = 'Bobby'; // no error; ignored
o.prop = 'Prop'; //no error; ignored

Object.freeze prevents objects from being changed.

Arrow Functions

const func = function() {
    return 9;
}

Example of an anonymous function.

const func = () => {
    return 9;
}

An anonymous function can be written using arrow function syntax.

const func = () => 9;

If there is only a return statement, then it can be rewritten like above.

const func = (num, mult) => num*mult;

Arrow function with arguments.

const func = num => num;

If there is only 1 argument, then the parentheses can be omitted.

const getMousePosition = (x, y) => ({ x, y });

A function that returns an object with properties x and y.

Rest Parameter

function howMany(...args) {
  return args.length;
}
console.log(howMany(0, 1, 2)); // 3

The rest parameter allows functions to take in a variable number of arguments, all of which are stored in an array.

Spread Operator

const arr = [6, 89, 3, 45];
const maximus = Math.max(...arr); // 89

The spread operator unpacks an array.

const arr = [6, 89, 3, 45];
const a = ...arr; // error
const b = [...arr]; // no error

It only works in-place, like as an argument to a function.

Destructuring Assignment (Objects)

const o = {
    name: 'Bob',
    age: 9
};

const {name, age} = o;

Similar to Python's tuple unpacking. The variables are the properties of the object.

const o = {
    name: 'Bob',
    age: 9
};

const {age: userAge} = o;

console.log(userAge) // 9

The properties can be assigned to new variables.

const user = {
  johnDoe: { 
    age: 34,
    email: 'johnDoe@freeCodeCamp.com'
  }
};

const { johnDoe: { age, email }} = user;

const { johnDoe: { age: userAge, email: userEmail }} = user;

Accessing properties of nested objects.

johnDoe is not a variable.

In a Function Argument

const p = ({ name, age, nationality, location }) => {
}

An object passed into a function will automatically be destructured.

Destructuring Assignment (Arrays)

const [a, b] = [1, 2, 3, 4, 5, 6];
console.log(a, b); // 1, 2

const [a, b,, c] = [1, 2, 3, 4, 5, 6];
console.log(a, b, c); // 1, 2, 4

The number of commas determines which value is grabbed.

const [a, b, ...arr] = [1, 2, 3, 4, 5, 7];
console.log(a, b); // 1, 2
console.log(arr); // [3, 4, 5, 7]

The rest parameter can be used to get the rest of the list.

Template Literals

const person = {
  name: "Big Tasty",
  age: 9
};

const greeting = `Hello, my name is ${person.name}!
I am ${person.age} years old.`;

console.log(greeting); // Hello, my name is Big Tasty!
// I am 9 years old.

Allows for multi-line strings and string interpolation.

Module Script

<script type="module" src="math.js"></script>

Making a script a module allows for parts of JavaScript files to be imported/exported to other JavaScript files.

Exporting

const add = (x, y) => x+y;
const sub = (x, y) => x-y;

export { add, sub };

Importing

import { add, sub } from './math.js';

add(5,3);
sub(5,4);

import * as myMathModule from './math.js';

myMathModule.add(2,3);
myMathModule.subtract(5,3);

Promises

Promises are useful for asynchronous tasks, like a server request.

const myPromise = new Promise((resolve, reject) => {
  if(condition here) {
    resolve("Promise was fulfilled");
  } else {
    reject("Promise was rejected");
  }
});

A promise has 3 states: pending, fulfilled, and rejected.

myPromise.then(result => {
    console.log(result);
}

then runs after resolve finishes.

result comes from the argument to resolve.

myPromise.catch(error => {
    console.log(error);
});

catch runs after reject finishes.

error comes from the argument to reject.
